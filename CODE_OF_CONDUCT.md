# Code of Conduct

## Our pledge

We want `rl-developer-memory` to be a respectful, constructive, and technically serious project.
Contributors, maintainers, and community members are expected to help create an environment that is welcoming, professional, and focused on learning and collaboration.

## Expected behavior

Examples of behavior that contribute to a positive environment include:
- being respectful in technical disagreement
- giving actionable review feedback
- assuming good intent while still being precise
- keeping criticism focused on code, design, or evidence
- respecting privacy, security, and responsible disclosure boundaries

## Unacceptable behavior

Examples of unacceptable behavior include:
- harassment, insults, or personal attacks
- publishing private information without permission
- abusive, discriminatory, or intentionally inflammatory language
- repeated bad-faith disruption of discussions, issues, or reviews
- requesting or sharing unsafe exploit details in public issue threads

## Scope

This code of conduct applies to project spaces such as:
- GitHub issues
- pull requests
- discussions
- review threads
- other official project communication channels

## Reporting

If you experience or witness unacceptable behavior, please contact the maintainer privately or use the project’s security/support channels when appropriate.
When reporting, include concrete context so the issue can be reviewed fairly.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, or contributions that violate this code of conduct.
Responses should be proportionate to the behavior and aimed at protecting the health of the project.
